### 1. 종료점 유형
종료점 유형에는 크게 총 3가지 유형이 있다. 반환값이 있는 경우, 상태 값 변경하는 종료점, 서드 파티를 호출하는 종료점.

1. 반환값이 있는 경우 
가장 기본적이고 테스트하기 쉬운 종료점이다. 함수나 메서드가 입력 값을 받아 내부적으로 연산을 수행한 뒤, 그 결과를 호출자에게 직접 돌려주는 경우
```javascript
// 테스트할 함수 (계산 결과 반환)
function add(a, b) {
  return a + b;
}

// 테스트 코드
const result = add(2, 3);
expect(result).toBe(5); // 반환값이 5인지 검증
```

2. 상태 값을 변경하는 경우
이는 시스템의 상태를 변경하는 경우이다. 실행 전후 시스템 내부의 데이터 모습이 달라지는 경우이다.
아래와 같은 상태 변경 케이스가 될 수 있다: 
```javascript
const user = { name: "민수", role: "USER" };

// 테스트할 함수 (반환값은 없지만 상태를 변경)
function promoteToAdmin(userObj) {
  userObj.role = "ADMIN";
}

// 테스트 코드
promoteToAdmin(user);
expect(user.role).toBe("ADMIN"); // 함수 실행 후의 상태(role) 검증
```

3. 서드파티를 호출하는 경우
함수가 자기 스스로 결과를 반환하거나 내부 상태를 반환하는 대신, 외부 시스템에 작업을 위임하고 통신하는 경우.
이 경우 외부 함수를 제어할 수 없으므로 가장 까다로운 테스트 경우.

```javascript
// Mock(가짜) 이메일 서비스 생성
const mockEmailService = {
  send: jest.fn() // 호출 여부를 기억하는 스파이 함수
};

// 테스트할 함수 (외부 서비스 호출)
function registerUser(email, emailService) {
  // ... 회원가입 로직 ...
  emailService.send(email, "가입을 환영합니다!"); // 서드파티 호출
}

// 테스트 코드
registerUser("test@example.com", mockEmailService);

// 실제 메일이 갔는지가 아니라, send 함수가 올바른 인자와 함께 1번 '호출'되었는지 검증
expect(mockEmailService.send).toHaveBeenCalledTimes(1);
expect(mockEmailService.send).toHaveBeenCalledWith("test@example.com", "가입을 환영합니다!");
```
